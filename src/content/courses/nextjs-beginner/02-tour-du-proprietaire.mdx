# Bienvenue dans les Coulisses ! 🕵️‍♀️

Maintenant que votre application `my-nexus-app` est créée, ouvrons le capot et regardons ce qu'il y a à l'intérieur. Ne vous inquiétez pas si tout ne semble pas clair immédiatement, nous y reviendrons !

Ouvrez le dossier `my-nexus-app` dans votre éditeur de code préféré (comme VS Code). Vous devriez voir une structure similaire à ceci :

<FileTree>
  my-nexus-app/
  ├── .next/           # Dossier de build (géré par Next.js) - Ne pas toucher !
  ├── node_modules/    # Dépendances du projet (géré par npm/yarn) - Ne pas toucher !
  ├── public/          # Fichiers statiques (images, polices...)
  │   ├── next.svg
  │   └── vercel.svg
  ├── src/             # ✨ C'est ici que vit VOTRE code ! ✨
  │   ├── app/         # ❤️ Cœur de l'application (Routing, Pages, Layouts)
  │   │   ├── favicon.ico
  │   │   ├── globals.css # Styles globaux (attention !)
  │   │   ├── layout.tsx  # Mise en page racine (TRÈS important)
  │   │   └── page.tsx    # UI de la page d'accueil ('/')
  │   ├── components/    # (Optionnel, bonne pratique) Pour vos composants réutilisables
  │   ├── lib/           # (Optionnel, bonne pratique) Pour les utilitaires, etc.
  │   └── ...            # D'autres dossiers que vous créerez !
  ├── .eslintrc.json   # Configuration ESLint (qualité du code)
  ├── .gitignore       # Fichiers/dossiers ignorés par Git
  ├── next.config.mjs  # Configuration de Next.js
  ├── package.json     # Infos projet, dépendances, scripts (comme 'npm run dev')
  ├── postcss.config.js # Configuration PostCSS (pour Tailwind)
  ├── tailwind.config.ts # Configuration Tailwind CSS
  └── tsconfig.json    # Configuration TypeScript
</FileTree>

<InfoTip type="info">
Les dossiers `.next` et `node_modules` sont gérés automatiquement. Concentrez-vous principalement sur les dossiers `public` et `src`.
</InfoTip>

## Les Zones Clés à Connaître :

1.  **`src/app/` : Le Cerveau du Routage et des Pages** ❤️
    *   C'est le **cœur** de votre application avec l'**App Router**.
    *   Chaque **dossier** ici représente un segment d'URL (une partie de l'adresse web).
    *   **`page.tsx`** : Définit l'interface utilisateur (le contenu) **unique** d'une route spécifique. La page d'accueil (`/`) est définie par `src/app/page.tsx`. Si vous créez `src/app/about/page.tsx`, cela créera la page `votresite.com/about`.
    *   **`layout.tsx`** : Définit une **mise en page partagée** pour une route et ses enfants. `src/app/layout.tsx` est la mise en page racine appliquée à *toutes* les pages. Utile pour le Header, Footer, Sidebar.
    *   **`globals.css`** : Fichier pour les styles CSS appliqués **globalement**. À utiliser avec parcimonie, préférez Tailwind ou les modules CSS.

2.  **`public/` : Les Ressources Accessibles** 🖼️
    *   Tout ce qui se trouve ici est servi **directement** à la racine de votre site.
    *   Idéal pour les images (`/mon-image.png`), les polices, les fichiers `robots.txt`, `sitemap.xml`, etc.
    *   Exemple : Si vous mettez `logo.svg` dans `public`, vous pouvez l'afficher avec `<img src="/logo.svg" />`.

3.  **`src/components/` (Bonne Pratique)** 🧱
    *   Ce dossier n'est pas créé par défaut mais c'est une **convention très répandue**.
    *   Rangez ici vos **composants React réutilisables** (Boutons, Cartes, Modales...) pour garder votre code `app/` propre.

4.  **`package.json` : La Carte d'Identité** 🆔
    *   Contient des informations sur votre projet (nom, version).
    *   Liste les **dépendances** (les bibliothèques externes que votre projet utilise, comme React, Next.js, Tailwind).
    *   Définit les **scripts** que vous pouvez lancer (comme `npm run dev`, `npm run build`).

5.  **`next.config.mjs` : Les Réglages Fins** ⚙️
    *   Fichier pour **configurer le comportement** de Next.js (optimisation d'images, redirections, variables d'environnement, etc.). On le touchera plus tard.
