# JSX, Késako ?

JSX est une extension de syntaxe pour JavaScript qui ressemble à du HTML mais avec toute la puissance de JavaScript.

## Pourquoi JSX ?

JSX nous permet d'écrire des éléments HTML dans JavaScript et de les placer dans le DOM sans avoir à utiliser `createElement()` ou `appendChild()`.

## Syntaxe de base

JSX ressemble à du HTML, mais il est en fait transformé en JavaScript lors de la compilation. Voici un exemple simple :

```jsx
const element = <h1>Bonjour, monde !</h1>;
```

## Expressions en JSX

Vous pouvez insérer n'importe quelle expression JavaScript valide dans JSX en l'entourant d'accolades `{}`.

```jsx
const nom = 'Expert Academy';
const element = <h1>Bonjour, {nom}</h1>;
```

## Attributs en JSX

Vous pouvez spécifier des attributs avec JSX, mais attention à ces différences par rapport au HTML :
- Les attributs utilisent la convention camelCase
- `class` devient `className` en JSX
- Les accolades `{}` sont utilisées pour les valeurs non-textuelles

## Points importants à retenir

1. JSX n'est pas du HTML, même s'il y ressemble
2. JSX est plus proche de JavaScript que de HTML
3. Vous pouvez utiliser toute la puissance de JavaScript dans votre JSX
4. JSX facilite la création d'interfaces utilisateur dans React
