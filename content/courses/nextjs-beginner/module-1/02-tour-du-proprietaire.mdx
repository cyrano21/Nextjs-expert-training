# Tour du Propriétaire : Comprendre la Structure d'un Projet Next.js

## Structure de Base

```
mon-projet/
├── app/
│   ├── layout.tsx
│   └── page.tsx
├── public/
├── components/
├── next.config.js
├── package.json
└── tsconfig.json
```

## Exploration Détaillée

### 📁 Dossier `app/`
Le cœur de votre application Next.js 13+
- `layout.tsx` : Mise en page commune
- `page.tsx` : Page d'accueil principale
- Routage basé sur la structure des fichiers

### 📁 Dossier `public/`
Ressources statiques
- Images
- Fichiers
- Polices
- Icônes

### 📁 Dossier `components/`
Composants réutilisables
- Boutons
- Cartes
- Éléments d'interface

### 📄 `next.config.js`
Configuration avancée de Next.js
- Plugins
- Redirections
- Configurations personnalisées

### 📄 `package.json`
Gestion des dépendances
- Scripts npm
- Versions des packages

### 📄 `tsconfig.json`
Configuration TypeScript
- Règles de typage
- Chemins d'importation

## Activité Interactive

### Défi : Identification des Rôles

Faites glisser chaque élément vers son rôle correct :
- `app/` → [Routage et pages]
- `public/` → [Ressources statiques]
- `components/` → [Composants réutilisables]

## Quiz de Validation

1. Quel fichier définit la mise en page globale ?
   - [ ] `page.tsx`
   - [x] `layout.tsx`
   - [ ] `next.config.js`

2. Où placeriez-vous un logo de votre site ?
   - [x] Dossier `public/`
   - [ ] Dossier `app/`
   - [ ] Dossier `components/`

## Conseils Pro

- Gardez `app/` organisé
- Utilisez des sous-dossiers pour les grandes applications
- Séparez la logique des composants
